# Описание модулей и интерфейсов

## 1. Общая структура исходного кода

Корневой каталог реализации: `src`.

- `core` - ядро инструкций и исключений;
- `memory` - Flash/SRAM и загрузка бинарника;
- `bus` - адресная маршрутизация;
- `nvic` - контроллер прерываний;
- `events` - очередь событий (event-driven периферия);
- `peripherals/timer` - модель TIM2;
- `peripherals/systick` - модель системного таймера SysTick;
- `peripherals/rcc` - модель reset/clock control;
- `peripherals/gpio` - модель GPIO портов;
- `peripherals/uart` - модель USART1;
- `debugger` - breakpoints;
- `profiler` - профилирование обработчиков и инструкций;
- `simulator` - оркестратор подсистем;
- `ui` - CLI;
- `gdb_stub` - сервер GDB RSP;
- `tests` - модульные и интеграционные испытания.

## 2. Модуль `common/status`

### Назначение

Единая система кодов возврата для всех подсистем.

### Перечень кодов

- `STATUS_OK`
- `STATUS_ERROR`
- `STATUS_INVALID_ADDRESS`
- `STATUS_INVALID_INSTRUCTION`
- `STATUS_BREAKPOINT_HIT`
- `STATUS_HALTED`

## 3. Модуль `memory`

### Назначение

Хранение и обслуживание доступа к областям памяти микроконтроллера.

### Данные

- Flash: `64 KB`;
- SRAM: `20 KB`.

### Публичный API

- `memory_init`, `memory_reset`;
- `memory_load_binary`;
- callbacks для Bus:
  - `memory_flash_read`, `memory_flash_write`;
  - `memory_sram_read`, `memory_sram_write`.

### Особенности

- Flash не очищается при reset;
- запись во Flash во время исполнения запрещена и возвращает ошибку.

## 4. Модуль `bus`

### Назначение

Единая точка доступа к memory-mapped пространству.

### Структуры

- `BusRegion`: `base`, `size`, `ctx`, `read`, `write`;
- `Bus`: массив регионов (`BUS_MAX_REGIONS = 16`).

### API

- `bus_init`;
- `bus_register_region`;
- `bus_read`;
- `bus_write`.

### Правила

- доступ к неотображенному адресу вызывает bus fault (диагностическое сообщение);
- размер доступа: `1`, `2`, `4` байта.

## 5. Модуль `nvic`

### Назначение

Программная модель контроллера прерываний.

### Параметры

- `NVIC_NUM_IRQ = 43`;
- приоритет: 8 бит, меньше значение - выше приоритет.

### API

- инициализация/сброс: `nvic_init`, `nvic_reset`;
- управление состояниями: `set/clear pending`, `enable/disable irq`, `set_priority`;
- выбор IRQ: `nvic_get_pending_irq`;
- жизненный цикл IRQ: `nvic_acknowledge`, `nvic_complete`.
- SysTick API: `nvic_set_systick_pending`, `nvic_enable_systick`, `nvic_get_systick_pending`.

### NVIC bus-периферия

`src/nvic/nvic_bus` предоставляет memory-mapped интерфейс к NVIC (`ISER/ICER/ISPR/ICPR/IABR/IPR`) для прошивок, работающих через стандартные регистры ядра.

## 5.1 Модуль `events/event_queue`

### Назначение

Планирование периферийных событий на заданный cycle.

### API

- `event_queue_init`;
- `event_queue_schedule`;
- `event_queue_dispatch`;
- `event_queue_peek`.

## 6. Модуль `core`

### Назначение

Исполнение инструкций и управление состоянием CPU.

### Состояние CPU

- регистры `R0-R15` (`SP`, `LR`, `PC`);
- `xPSR` и флаги `N/Z/C/V`;
- `thumb_mode`, `interruptible`;
- `current_irq`;
- `cycles`.

### API

- `core_init`;
- `core_reset`;
- `core_step`;
- `core_get_state`.

### Внутренние блоки

- helpers обновления флагов;
- блок `enter_exception/exit_exception`;
- таблица декодирования инструкций;
- отдельная обработка части 32-битных инструкций.

### Поддерживаемые классы инструкций (основные)

- арифметика и логика;
- переходы;
- доступ к памяти;
- стековые операции;
- базовые Thumb-2 операции (`BL`, `MOVW`, `MOVT`, часть data processing immediate).

## 7. Модуль `peripherals/timer`

### Назначение

Модель таймера TIM2.

### Регистры

- `CR1`, `DIER`, `SR`, `CNT`, `PSC`, `ARR`.

### Поведение

- счетчик увеличивается при `CEN=1`;
- предделитель задается `PSC`;
- при переполнении выставляется `UIF`;
- при `UIE=1` формируется pending IRQ в NVIC.

### API

- `timer_init`;
- `timer_as_peripheral`;
- `timer_read/write/tick/reset`.

## 8. Модуль `peripherals/uart`

### Назначение

Модель UART USART1 с передачей/приемом и IRQ.

### Регистры

- `SR`, `DR`, `BRR`, `CR1`.

### Состояние

- TX-механизм с отложенной передачей в `tick`;
- RX кольцевой буфер (`UART_RX_BUFFER_SIZE = 16`);
- callback передачи символов во внешний канал (по умолчанию stdout).

### API

- `uart_init`, `uart_reset`;
- `uart_set_output`;
- `uart_incoming_char`;
- `uart_as_peripheral`;
- `uart_read/write/tick`.

## 8.1 Модуль `peripherals/uart/uart_logger`

### Назначение

Кольцевой логгер UART TX/RX с генерацией ASCII/SVG/HTML-диаграммы.

### API

- `uart_logger_init`, `uart_logger_tick`, `uart_logger_clear`;
- `uart_logger_log_tx`, `uart_logger_log_rx`;
- `uart_logger_generate_diagram`, `uart_logger_write_svg`, `uart_logger_write_html`.

## 8.2 Модуль `peripherals/systick`

### Назначение

Модель системного таймера Cortex-M3.

### Особенности

- memory-mapped по адресу `0xE000E010`;
- работает в связке с `EventQueue`;
- поднимает SysTick exception через NVIC при `TICKINT`.

## 8.3 Модуль `peripherals/rcc`

### Назначение

Минимальная совместимая модель RCC для корректного старта firmware.

### Особенности

- ready-флаги синхронизируются с enable-битами (stub-модель);
- clock tree не моделируется в аппаратной точности.

## 8.4 Модуль `peripherals/gpio`

### Назначение

Модель GPIO портов A/B/C.

### Особенности

- поддержка `CRL/CRH/IDR/ODR/BSRR/BRR/LCKR`;
- расчет IDR с учетом режима пина (input/output);
- вспомогательные функции внешнего воздействия (`gpio_set_pin`).

## 9. Модуль `debugger`

### Назначение

Обслуживание программных breakpoints.

### Параметры

- максимум `64` breakpoint-адреса.

### API

- `debugger_init`;
- `debugger_add_breakpoint`;
- `debugger_remove_breakpoint`;
- `debugger_check`;
- `debugger_list`.

## 10. Модуль `simulator`

### Назначение

Оркестратор всей модели микроконтроллера.

### Функции

- хранит инстансы подсистем;
- инициализирует и связывает модули;
- управляет режимами `run/step/reset/halt`;
- обеспечивает загрузку прошивки;
- поддерживает подключение дополнительных периферий.

### API

- `simulator_init`, `simulator_reset`;
- `simulator_step`, `simulator_run`, `simulator_halt`;
- `simulator_load`;
- `simulator_add_peripheral`.

## 11. Модуль `ui`

### Назначение

Интерактивное управление симулятором через консоль.

### Команды

- `help`, `load`, `run`, `stop`, `step`, `reset`;
- `reg`, `mem`;
- `break`, `delete`;
- `uart`, `diagram`, `diagram-save`, `profile`;
- `quit`.

## 12. Модуль `gdb_stub`

### Назначение

Удаленная отладка через GDB Remote Serial Protocol.

### API

- `gdb_stub_init`;
- `gdb_stub_run` (блокирующий цикл);
- `gdb_stub_close`.

### Поддерживаемые команды RSP

- `?`, `g/G`, `p/P`, `m/M`, `c`, `s`, `Z/z`;
- `qSupported`, `qRcmd`, `qXfer:features:read`;
- thread queries (`qfThreadInfo`, `qsThreadInfo`, `H`, `T`);
- `D`, `k`.

## 13. Модуль `tests`

### Назначение

Проверка корректности модулей и интеграции.

### Покрываемые направления

- NVIC;
- Memory;
- Bus;
- Timer;
- SysTick;
- RCC;
- GPIO;
- UART;
- Core;
- Debugger;
- интеграционные сценарии (IRQ path, breakpoint halt, UART output и расширенные сценарии после добавления новых периферий).
