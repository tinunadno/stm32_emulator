# Программа и методика испытаний

## 1. Назначение документа

Документ определяет состав, порядок и критерии испытаний симулятора STM32F103C8T6 для подтверждения соответствия функциональным требованиям.

## 2. Объект испытаний

Программный комплекс, включающий:

- исполняемый файл `stm32sim`;
- тестовый раннер `test_runner`;
- подсистемы CPU, Memory, Bus, NVIC (+ NVIC bus), EventQueue, Timer, SysTick, RCC, GPIO, UART, Debugger, CLI, GDB Stub.

## 3. Цели испытаний

- подтвердить корректность базовых функций симуляции;
- подтвердить корректность взаимодействия модулей в сквозных сценариях;
- проверить устойчивость системы к ошибочным воздействиям;
- обеспечить воспроизводимость результатов.

## 4. Виды испытаний

### 4.1 Модульные испытания

Покрывают отдельные подсистемы:

- `test_nvic`
- `test_memory`
- `test_bus`
- `test_timer`
- `test_systick`
- `test_rcc`
- `test_gpio`
- `test_uart`
- `test_core`
- `test_debugger`

### 4.2 Интеграционные испытания

`test_integration` включает:

- путь `Timer -> NVIC -> Core exception handler`;
- остановку на breakpoint;
- передачу UART-символа через callback.

### 4.3 Ручные приемочные испытания

- CLI-операции;
- запуск прошивки из бинарника;
- подключение GDB;
- пошаговое выполнение и breakpoints из GDB.

## 5. Условия и средства проведения

- среда сборки: `gcc`, `make`;
- запуск тестов: `make test` в `src`;
- логи тестов выводятся в stdout;
- успешность определяется кодом завершения процесса.

## 6. Методика автоматических испытаний

1. Выполнить `make clean`.
2. Выполнить `make`.
3. Выполнить `make test`.
4. Убедиться, что итог тестов: `0 failed`.
5. Зафиксировать результаты в отчете испытаний.

## 7. Методика ручных испытаний (CLI)

### Сценарий 1: базовая загрузка и запуск

1. Запустить `./stm32sim firmware.bin`.
2. Выполнить `reg` и убедиться в валидных `SP/PC`.
3. Выполнить `step 10`.
4. Выполнить `run` и `stop`.

**Критерий приемки:** нет аварий, команды выполняются штатно.

### Сценарий 2: контроль breakpoints

1. `break <адрес>`
2. `run`
3. Проверить остановку на указанном `PC`.
4. `delete <адрес>`, повторить запуск.

**Критерий приемки:** остановка и снятие точки останова корректны.

### Сценарий 3: проверка памяти

1. `mem 0x20000000 64`
2. `mem 0x08000000 64`
3. Провести проверку обращения к заведомо невалидному адресу.

**Критерий приемки:** корректный дамп в валидных диапазонах, диагностическое сообщение при invalid address.

### Сценарий 4: UART-диаграмма и профиль

1. Выполнить UART-обмен (`uart X`, запуск/шаги).
2. Выполнить `diagram`.
3. Выполнить `diagram-save`.
4. Выполнить `profile`.

**Критерий приемки:** диаграмма отображает события TX/RX, файл диаграммы сохраняется, профиль выводится без ошибок.

## 8. Методика ручных испытаний (GDB)

### Сценарий 5: подключение и чтение регистров

1. `./stm32sim firmware.bin --gdb 3333`
2. Подключить GDB к `:3333`.
3. Выполнить `info registers`.

**Критерий приемки:** корректное чтение регистров и отсутствие обрыва сессии.

### Сценарий 6: выполнение и останов

1. Поставить breakpoint в GDB.
2. Выполнить `continue`.
3. Выполнить `stepi`.
4. Прервать `Ctrl+C`.

**Критерий приемки:** срабатывание breakpoint, пошаговое выполнение, корректный stop signal.

## 9. Негативные сценарии

- запуск с несуществующим бинарником;
- запуск с некорректным портом GDB;
- попытка записи во Flash во время исполнения;
- чтение/запись по неотображенному адресу Bus;
- переполнение буфера RX UART;
- доступ к неизвестным смещениям RCC/GPIO/SysTick/NVIC bus.

**Критерий приемки:** отсутствие падения процесса, корректный код ошибки/диагностика.

## 10. Критерии приемки системы

Система считается прошедшей испытания при одновременном выполнении условий:

- `make test` завершается без падений и без failed тестов;
- ручные сценарии CLI и GDB выполнены успешно;
- дефекты уровня "критический" отсутствуют;
- комплект документации соответствует [[00_Паспорт_комплекта_документации|паспорту комплекта]].

## 11. Отчетные материалы

По итогам испытаний формируются:

- протокол автотестов;
- протокол ручной проверки CLI/GDB;
- перечень выявленных дефектов (если есть) и их статус;
- заключение о готовности к демонстрации/защите.
