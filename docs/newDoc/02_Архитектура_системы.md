# Архитектура системы

## 1. Архитектурный подход

Симулятор построен как модульная система с централизованной оркестрацией через структуру `Simulator`.

Ключевые принципы:

- явное разделение ответственности модулей;
- обмен через четкие интерфейсы (функции + структуры состояния);
- единый цикл моделирования: `tick -> execute -> debug checks`;
- адресная интеграция компонентов через системную шину.

## 2. Состав подсистем

- **Core:** исполнение инструкций ARM Thumb/Thumb-2, обработка исключений.
- **Memory:** реализация Flash и SRAM, загрузка бинарного образа.
- **Bus:** маршрутизация всех операций чтения/записи в memory-mapped пространство.
- **NVIC:** учет pending/active/enabled, приоритетная выборка IRQ + SysTick exception.
- **Events:** очередь событий для cycle-based периферии.
- **Peripherals:** TIM2, SysTick, RCC, GPIOA/B/C, UART USART1.
- **Debugger:** программные breakpoints.
- **Profiler:** сбор статистики обработчиков/инструкций.
- **UI:** интерактивный CLI.
- **GDB Stub:** удаленный интерфейс отладки по RSP.

## 3. Схема взаимодействия модулей

1. `main` создает и инициализирует `Simulator`.
2. `simulator_init` поднимает Memory/NVIC/Bus, регистрирует регионы памяти и периферии, инициализирует Core и Debugger.
3. При запуске шага:
   - вызываются `tick` всех зарегистрированных периферий;
   - выполняется одна инструкция `core_step`;
   - проверяются breakpoints.
4. При наличии pending IRQ/исключений ядро запрашивает NVIC и выполняет вход в обработчик.

## 4. Адресное пространство (реализованный минимум)

| Блок | Базовый адрес | Размер | Назначение |
|---|---:|---:|---|
| Flash alias | `0x00000000` | `64 KB` | таблица векторов/код |
| Flash main | `0x08000000` | `64 KB` | основная программа |
| SRAM | `0x20000000` | `20 KB` | данные/стек |
| TIM2 | `0x40000000` | `0x400` | таймер |
| GPIOA | `0x40010800` | `0x400` | GPIO порт A |
| GPIOB | `0x40010C00` | `0x400` | GPIO порт B |
| GPIOC | `0x40011000` | `0x400` | GPIO порт C |
| USART1 | `0x40013800` | `0x400` | UART |
| RCC | `0x40021000` | `0x400` | reset/clock control |
| SysTick | `0xE000E010` | `0x10` | системный таймер |
| NVIC bus | `0xE000E100` | `0x400` | регистры управления IRQ |

## 5. Цикл исполнения

### 5.1 Инициализация

- `memory_init`, `nvic_init`, `bus_init`, `event_queue_init`;
- `bus_register_region` для Flash alias, Flash, SRAM;
- `timer_init`, `systick_init`, `rcc_init`, `gpio_init`, `uart_init`;
- подключение `uart_logger` и `profiler`;
- регистрация периферии в Bus и в списке tickable устройств;
- `core_init`, `debugger_init`.

### 5.2 Сброс

- reset периферии;
- `nvic_reset`, `memory_reset`, `core_reset`;
- очистка очереди событий;
- загрузка `SP` и `PC` из векторной таблицы;
- сброс флагов состояния симулятора.

### 5.3 Один шаг моделирования

1. Инкремент глобального счетчика `cycle`.
2. `tick` polling-устройств (включая UART/Timer).
3. `event_queue_dispatch` для event-driven периферии (например SysTick/Timer).
4. Fetch/decode/execute одной инструкции в Core.
5. Обработка IRQ и core exceptions (в т.ч. SysTick) при необходимости.
6. Проверка попадания на breakpoint.

## 6. Подход к расширяемости

Расширение реализуется через интерфейс `Peripheral`:

- `read/write` для доступа через MMIO;
- `tick` для поведения во времени;
- `reset` для системного сброса.

Новый модуль добавляется без изменения ядра CPU:

1. создать состояние и callbacks;
2. сформировать `Peripheral`;
3. зарегистрировать через `simulator_add_peripheral`.

## 7. Подход к отладке

### 7.1 Локальная отладка

CLI поддерживает:

- `step`, `run`, `stop`, `reset`;
- просмотр регистров (`reg`);
- просмотр памяти (`mem`);
- управление breakpoints (`break`, `delete`);
- визуализацию UART активности (`diagram`, `diagram-save`);
- вывод профиля IRQ/инструкций (`profile`).

### 7.2 Удаленная отладка

RSP-сервер поддерживает:

- чтение/запись регистров (`g/G`, `p/P`);
- чтение/запись памяти (`m/M`);
- управление выполнением (`c`, `s`, Ctrl+C);
- breakpoints (`Z0`/`z0`);
- описание цели через `qXfer:features:read:target.xml`.

## 8. Ограничения текущей архитектуры

- покрыто подмножество инструкций, а не весь ISA Cortex-M3;
- реализация тайминга упрощенная;
- модель периферии (RCC/GPIO/SysTick) частично stub-уровня;
- отсутствует загрузчик ELF (доступна загрузка raw binary);
- отсутствует полноценная система логирования уровней.
