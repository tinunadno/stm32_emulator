# Руководство пользователя

## 1. Назначение документа

Документ определяет порядок запуска и эксплуатации симулятора `STM32F103C8T6` в пользовательских сценариях: CLI-режим и режим удаленной отладки через GDB.

## 2. Условия применения

Для работы требуется:

- исходный код проекта;
- `gcc` и `make`;
- терминал командной строки;
- при необходимости удаленной отладки - `arm-none-eabi-gdb`.

## 3. Сборка

Из каталога `src`:

```bash
make
```

Результат: исполняемый файл `stm32sim`.

## 4. Запуск программы

## 4.1 Базовый запуск

```bash
./stm32sim
```

Запускается CLI без предварительной загрузки бинарника.

## 4.2 Запуск с бинарной прошивкой

```bash
./stm32sim firmware.bin
```

При успешной загрузке выполняется reset и доступно интерактивное управление.

## 4.3 Запуск в режиме GDB

```bash
./stm32sim firmware.bin --gdb
```

Или с явным портом:

```bash
./stm32sim firmware.bin --gdb 4444
```

## 5. Работа в CLI

### 5.1 Общие правила

- каждая команда вводится с новой строки;
- параметры адресов допускают hex (`0x...`) и decimal;
- ошибки команд не завершают приложение.

### 5.2 Справочник команд

- `help` - список команд;
- `load <path>` - загрузка бинарника в Flash;
- `run` - непрерывный запуск до остановки;
- `stop` - принудительная остановка;
- `step [N]` - выполнение N инструкций;
- `reset` - системный reset;
- `reg` - отображение регистров CPU;
- `mem <addr> [count]` - дамп памяти;
- `break <addr>` - установка breakpoint;
- `delete <addr>` - удаление breakpoint;
- `uart <char>` - подача символа на вход UART;
- `quit` - завершение работы.

### 5.3 Типовой сценарий

1. `load firmware.bin`
2. `reg` (контроль стартовых SP/PC)
3. `break 0x08000086`
4. `run`
5. `step 5`
6. `mem 0x20000000 64`

## 6. Работа через GDB

## 6.1 Подключение

```bash
arm-none-eabi-gdb firmware.elf
(gdb) target remote :3333
```

## 6.2 Рекомендуемые команды GDB

- `monitor reset halt`
- `info registers`
- `x/16wx 0x20000000`
- `break *0x08000080`
- `continue`
- `stepi`

## 6.3 Особенности режима

- симулятор однопоточен;
- breakpoints программные;
- поддерживается прерывание выполнения через `Ctrl+C`;
- часть monitor-команд принимается как совместимые (`halt`, `reset`, `reset halt`).

## 7. Возможные проблемы и действия пользователя

### 7.1 Бинарник не загружается

Проверить:

- существование файла;
- права доступа;
- что файл не пустой.

### 7.2 Ошибка "Invalid port"

Использовать порт в диапазоне `1..65535`.

### 7.3 Bus fault в CLI

Причина: чтение/запись по неотображенному адресу. Требуется проверить адресный диапазон.

### 7.4 Программа "зависла"

Возможные случаи:

- активный бесконечный цикл в firmware;
- ожидание события прерывания;
- остановка на breakpoint.

Рекомендуется применить `stop`, затем `reg` и `mem`.

## 8. Требования безопасности и сохранности данных

- симулятор не должен использоваться для выполнения непроверенных внешних бинарников в доверенной инфраструктуре;
- пользователь несет ответственность за содержимое загружаемой прошивки;
- запись в хост-файловую систему в штатном режиме симуляции не производится.

## 9. Завершение работы

Корректное завершение:

- в CLI - команда `quit`;
- в GDB режиме - `detach`/`quit` со стороны отладчика или остановка процесса симулятора.
