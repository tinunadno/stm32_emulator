# Компонент: Simulator Worker

Воркер — процесс, который выполняет симуляцию прошивок и, при необходимости, поднимает GDB-сервер. 

## Основные свойства

| Аспект        | Значение                              |
|--------------|---------------------------------------|
| Язык         | Go 1.21+                             |
| GDB-порты    | Динамический диапазон (например 1234–1244) |
| Concurrency  | 1 задание на воркер в debug-режиме    |

## Обязанности

- Регулярный heartbeat в KeyDB, чтобы другие компоненты знали о его активности. 
- Получение задач из очереди `jobs:pending` через `BRPOPLPUSH`. 
- Выбор режима работы:
  - обычная симуляция (без GDB);
  - debug-симуляция (с GDB-сервером и увеличенным таймаутом). 
- Публикация событий в каналы `events:job:{id}` (статусы, логи, телеметрия). 
- Обработка команд остановки и авто-таймаутов debug-сессий. 

## Реализация debug-режима

- Воркер выбирает свободный порт из диапазона и поднимает GDB-сервер. 
- Обновляет KeyDB (`job:{id}`) полями `gdb_port`, `gdb_host`, `gdb_connected=false`. 
- Публикует событие о готовности GDB с параметрами подключения. 
- Ожидает подключение GDB-клиента до таймаута, после чего запускает симуляцию. 

Подробнее о режимах смотрите в [[Debug Flow/Worker Modes|Worker Modes]].
