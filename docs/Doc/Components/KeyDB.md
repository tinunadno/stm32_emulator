# Компонент: KeyDB

KeyDB используется как шина данных: хранит состояние задач, очереди и события. 

## Конфигурация

| Аспект       | Значение  |
|-------------|-----------|
| Версия      | 6.2+      |
| Порт        | 6379      |
| Persistence | AOF + RDB (опционально) |

## Основные структуры

- Hash `job:{job_id}` — метаданные задания, включая debug-поля. 
- List `jobs:pending` — очередь задач, ожидающих обработки воркером. 
- List `jobs:processing` — задания, которые сейчас выполняются. 
- Pub/Sub каналы `events:job:{job_id}` — события симуляции и статусы. 

Дополнительные поля для GDB описаны в [[KeyDB Protocols/Job Metadata|Job Metadata]].
