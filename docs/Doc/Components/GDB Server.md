# Компонент: GDB Server (gdbstub)

GDB-сервер реализует протокол GDB Remote Serial Protocol (RSP) и связывается с симулятором STM32. 

## Варианты реализации

- Встроенный gdbstub в код симулятора (на Go или другом языке). 
- Использование внешнего gdbserver, если backend — QEMU или другая среда, поддерживающая RSP. 

## Основные задачи

- Поднять TCP-сервер на указанном порту и ожидать подключения GDB-клиента. 
- Реализовать разбор пакетов GDB RSP (`$...#xx`) и ответов. 
- Пробрасывать операции чтения/записи регистров и памяти в симулятор. 
- Управлять выполнением симуляции (continue, step, breakpoints). 

Примеры минимальной реализации gdbstub и обработчика команд приведены в [[Implementation Details/GDB RSP|GDB RSP и gdbstub]].
