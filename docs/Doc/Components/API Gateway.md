# Компонент: API Gateway

API Gateway — внешний вход в систему. Предоставляет REST API и SSE-потоки для клиентов. 

## Стек и конфигурация

| Аспект       | Значение                |
|-------------|-------------------------|
| Язык        | Go 1.21+               |
| Фреймворк   | Chi или net/http       |
| Порты       | 8080 (REST), 8081 (метрики) |
| Состояние   | Stateless, масштабируется горизонтально |

## Обязанности

- Аутентификация по API Key и определение `user_id`. 
- Валидация входных данных, включая `binary_b64`, `debug`, `timeout_seconds`. 
- Сохранение записей в PostgreSQL: задание, статус, GDB-метаданные. 
- Публикация задач в KeyDB: создание `job:{id}` и добавление ID в `jobs:pending`. 
- Организация SSE-подписок и трансляция событий из KeyDB клиентам. 
- Эндпоинт `GET /v1/jobs/{job_id}/gdb-info` для получения информации о GDB-сервере. 

## Важно

Gateway **не** проксирует GDB-протокол, соединение GDB-клиент ↔ воркер всегда прямое. 

Подробности REST API — в [[External API/Jobs API|Jobs API]].  
Описание SSE-событий — в [[External API/SSE Events|SSE Events]].  
Эндпоинт для GDB — в [[External API/GDB Info Endpoint|GDB Info Endpoint]].
