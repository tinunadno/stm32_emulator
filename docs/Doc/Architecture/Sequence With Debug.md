# Диаграмма последовательности (debug)

Диаграмма последовательности отражает взаимодействие Клиента, Gateway, KeyDB, Worker и GDB при запуске debug-задания. 

## Текстовая диаграмма

```text
Клиент          Gateway         KeyDB          Worker          GDB
  |                |              |              |              |
  | POST /jobs     |              |              |              |
  | (debug=true)   |              |              |              |
  |--------------->|              |              |              |
  |                | HSET job:... |              |              |
  |                |------------->|              |              |
  |                | LPUSH pending|              |              |
  |                |------------->|              |              |
  | 202 Accepted   |              |              |              |
  |<---------------|              |              |              |
  |                |              | BRPOPLPUSH   |              |
  |                |              |<-------------|              |
  |                |              |--------------|              |
  |                |              | job:...      |              |
  |                |              |              |              |
  |                |              |              | Start GDB    |
  |                |              |              | stub on port |
  |                |              |              |=============>|
  |                |              |              |              |
  |                |              | HSET gdb_port|              |
  |                |              |<-------------|              |
  |                |              |--------------|              |
  |                | PUBLISH event|              |              |
  |                |<-------------|              |              |
  | SSE: gdb_info  |              |              |              |
  |<---------------|              |              |              |
  |                |              |              |              |
  |                |              |              |              | target remote
  |                |              |              |<-------------|
  |                |              |              |--------------|
  |                |              |              | GDB protocol |
  |                |              |              |<------------>|
  |                |              |              |              |
  |                |              |              | Simulation   |
  |                |              |              | controlled   |
  |                |              |              | by GDB       |
  |                |              |              |              |
```

Эта последовательность описывает создание задания, запуск GDB-stub на воркере, публикацию информации о порте и последующее управление симуляцией через GDB.
