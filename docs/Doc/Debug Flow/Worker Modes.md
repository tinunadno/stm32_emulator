# Режимы работы воркера

Worker имеет два режима работы: обычный и debug. 

## Normal Mode

- Запускает симуляцию без GDB. 
- Использует стандартный таймаут задания. 
- Публикует логи и телеметрию. 

## Debug Mode

- Запускает симуляцию с GDB-сервером на выделенном порту. 
- Ожидает подключения GDB-клиента, возможно с увеличенным таймаутом. 
- Позволяет управление симуляцией через GDB (breakpoint, step, continue). 
- Может иметь отдельный idle-таймаут, при котором симуляция останавливается при отсутствии активности. 

Схема:

```text
Worker
 ├─ Normal Mode
 │   ├─ Запуск симуляции
 │   └─ Публикация логов
 └─ Debug Mode
     ├─ Поднятие GDB-сервера
     ├─ Ожидание подключения GDB
     └─ Запуск симуляции под управлением GDB
```

Подробный сценарий debug-сессии — в [[Debug Flow/GDB Session Lifecycle|GDB Session Lifecycle]].
