# Жизненный цикл GDB-сессии

Отправка debug-задания и интерактивная отладка прошивки через GDB. 

## Шаги с точки зрения разработчика

1. Отправить задание `POST /v1/jobs` с `debug: true`. 
2. Получить `job_id` и подписаться на SSE или опрашивать `/gdb-info`. 
3. Дождаться статуса `running` и параметров GDB (`host`, `port`). 
4. Запустить локальный GDB с ELF и выполнить `target remote <host>:<port>`. 
5. Отлаживать с помощью `break`, `step`, `next`, `continue`, `print`, `info registers`, `x` и т.п. 
6. Завершить debug-сессию командами `detach` и `quit`. 

## Внутренние шаги системы

- Воркер поднимает GDB-сервер и обновляет записи в KeyDB и PostgreSQL. 
- При подключении клиента обновляется `gdb_connected=true`, логируется IP и время в `debug_sessions`. 
- Метрики и трейсы собираются для анализа длительности и нагрузки GDB-сессий. 
- При достижении лимита по времени или простоя симуляция завершается. 

Пример GDB-сессии можно посмотреть в [[How-To/Connect GDB|Connect GDB]].
